# Productization, App, Cloud, and Community Roadmap

本文档描述 `04_camera_pet_battle` 的产品化方向。当前固件必须保持 CoreS3 本地离线可运行；App、云端、大模型、SD 卡和社区能力均为未来预留方向，不是当前运行依赖。

## 当前边界

当前已经具备的能力：

- 拍照捕捉宠物。
- 本地轻量限定类别识别，用于从有限特征中生成宠物，不是通用视觉大模型。
- 背包通过 ESP32 `Preferences` 持久化。
- 对战通过 Wi-Fi AP + UDP 进行本地通信。
- 音效可静音。
- 不依赖 SD 卡。
- 不依赖外网或云服务。
- 无语音识别。

当前明确不足：

- 本地识别类别有限，不能承诺识别任意物体、材质或场景。
- 当前不上传图片，也不调用商业大模型。
- 当前没有 App 客户端。
- 当前没有 SD 卡样本、日志、截图或模型文件管理。
- 当前没有账号、社区、排行榜或云端宠物同步。
- 当前不能承诺 CoreS3 本地运行 YOLO、CLIP 或视觉大模型。

## 产品目标

短期目标是把 CoreS3 固件打磨成可演示、可测试、可采样的本地玩具原型：

- 用户可以拍照捕捉宠物。
- 用户可以查看背包和宠物详情。
- 用户可以进行本地对战。
- 用户可以导出日志和样本，支持后续训练。
- 用户可以清楚理解当前识别能力边界。

中长期目标是围绕 App、数据集、云端推理和社区生态扩展体验：

- App 提供更清晰的宠物卡片、战绩、友情值和连接教程。
- 电脑端工具优先支持串口连接、日志导出和样本采集。
- 云端可选增强识别和文案生成，但不能成为固件运行前提。
- SD 卡可选保存样本、日志、截图、宠物记录和模型文件。
- 社区能力可在未来支持宠物分享、图鉴、对战记录和活动。

建议先做 iPhone App 产品规格，但实现优先级仍以电脑端 MVP 为先，因为 USB 串口调试、日志导出和样本采集更容易闭环。

  - 友情值
  - 战绩
  - 备注或昵称
- 捕捉
  - 触发拍照
  - 查看缩略图或识别摘要

  - 保存样本
- 对战
  - 本机角色
  - 对手状态
  - 回合结果
  - 对战日志
- 数据导出
  - 导出日志
  - 导出样本索引
  - 导出宠物快照
- 连接教程
  - USB 串口连接
  - Wi-Fi AP + UDP 对战连接
  - 常见问题
- 设置
  - 静音开关说明
  - 数据权限说明
  - 云端增强开关预留

## 本地 MVP 功能列表

本地 MVP 不依赖云端、不依赖 SD 卡、不依赖网络。

必须包含：

- 查看背包。
- 查看宠物详情。
- 查看宠物卡片。
- 查看战绩。
- 查看友情值。
- 触发拍照。
- 查看本地识别结果。
- 查看对战状态。
- 导出日志。
- 导出样本。
- 提供连接教程。

建议先做电脑端 MVP：

- Python + PySide：适合快速串口工具、表格和日志界面。
- Tauri：适合轻量桌面 App，后续可复用 Web UI。
- Electron：生态成熟，但体积较大。

手机端后续路线：

- Flutter：跨平台一致性较强。
- React Native：适合已有 Web/React 团队。
- 原生 Android：适合先验证 BLE、USB OTG 或局域网发现。
- iOS 原生：适合最终 iPhone 体验，但早期调试成本较高。

## 通信路线

第一阶段：USB 串口。

- 目标：连接稳定、便于调试、便于导出日志和样本。
- 数据：设备状态、背包摘要、宠物详情、识别结果、对战状态、日志。
- 触发：拍照、导出、清空日志、读取快照。

第二阶段：Wi-Fi HTTP/UDP。

- HTTP 用于 App 查询状态、读取背包、拉取日志和触发拍照。
- UDP 继续用于本地对战，保持现有 Wi-Fi AP + UDP 设计。
- 不要求外网；局域网或设备 AP 即可。

第三阶段：BLE 或局域网发现。

- BLE 用于低功耗连接、配对和基础状态读取。
- 局域网发现用于多个设备或桌面工具自动发现。
- 该阶段需要单独评估包大小、连接稳定性和 CoreS3 资源占用。

## 云端大模型规划

云端能力必须是可选增强，不得成为当前固件运行前提。

约束：

- 不写死任何 API key。
- 不绑定具体商业大模型。
- 不上传用户图片，除非未来 UI 明确征得用户同意。
- 不影响离线捕捉、背包和对战。
- 固件侧保留超时和降级策略，云端失败时继续使用本地结果。

云端输入可包括：

- 图片或缩略图。
- `ImageTraits`。
- `RecognitionResult`。
- 设备 ID。
- 背包上下文。

云端输出可包括：

- `objectLabel`
- `materialLabel`
- `elementHint`
- `speciesBias`
- `confidence`
- `petName`
- `evolutionHint`
- `flavorText`

固件预留接口复用方向：

- `fetch_remote_pet_hint`：获取云端可选提示。
- `merge_generation_inputs`：合并本地识别、云端提示和背包上下文。
- `save_pet_snapshot`：保存生成后的宠物快照，便于导出和复现。

## SD 卡规划

SD 卡是可选存储扩展，不是当前运行依赖。

用途：

- 保存样本。
- 保存日志。
- 保存截图。
- 保存宠物记录。
- 保存模型文件或训练导出产物。

注意：

- SD 卡不提升模型推理速度。
- 无 SD 卡时系统仍必须可运行。
- 固件应把 SD 卡写入失败视为可恢复事件，不影响捕捉、背包和对战主流程。

建议目录：

```text
/pets/
/captures/
/samples/
/logs/
/models/
```

## 数据集与模型训练规划

样本采集先规划 8 类：

- `plant_leaf`：植物叶片。
- `food_fruit`：食物或水果。
- `paper_book`：纸张或书本。
- `electronics_screen`：电子设备或屏幕。
- `metal_key_coin`：金属、钥匙或硬币。
- `fabric_cloth`：布料或衣物。
- `cup_bottle_water`：杯子、瓶子或水容器。
- `toy_figure`：玩具或手办。

采集字段建议：

- 样本 ID。
- 采集时间。
- 设备 ID。
- 缩略图或图片路径。
- `ImageTraits`。
- `RecognitionResult`。
- 人工标签。
- 是否可用于训练。

训练路线：

- 电脑端训练优先。
- 训练脚本读取样本索引和人工标签。
- 输出轻量特征模型。
- 导出 `vision_model_data.h`。
- 固件只加载导出的静态模型数据，不在 CoreS3 上训练。

可评估方案：

- TFLite Micro。
- Edge Impulse。
- ESP-DL。

不承诺事项：

- 不承诺 CoreS3 本地跑 YOLO。
- 不承诺 CoreS3 本地跑 CLIP。
- 不承诺 CoreS3 本地跑视觉大模型。

## 未来社区规划

社区能力放在本地 MVP 和可选云端增强之后。

候选能力：

- 宠物图鉴分享。
- 战报分享。
- 主题活动。
- 用户上传样本集，需明确授权。
- 公开模型版本说明。
- 宠物命名和描述模板。

隐私边界：

- 默认不上传图片。
- 上传样本必须由用户主动触发。
- 云端 API key 只能由用户配置或后端安全管理，不能写死在固件或公开仓库。

## 协议与接口建议

不修改 `BattlePetPacket`。对战协议继续保持 Wi-Fi AP + UDP。

如果未来需要 App 查询状态，建议新增独立的调试/管理协议，不复用或扩展现有对战 UDP 包：

- `GET_STATUS`：读取设备状态、当前屏幕、静音状态。
- `GET_BACKPACK_SUMMARY`：读取背包摘要。
- `GET_PET_DETAIL`：读取指定宠物详情。
- `CAPTURE_ONCE`：触发一次拍照。
- `GET_LAST_RECOGNITION`：读取最近识别结果。
- `GET_BATTLE_STATUS`：读取对战状态。
- `EXPORT_LOG`：导出日志。
- `EXPORT_SAMPLE_INDEX`：导出样本索引。

接口变更建议：

- 新增 App 管理协议时，应单独定义包头、版本和命令枚举。
- 不修改 `BattlePetPacket`。
- 不重新定义 `PetGenes`、`SavedPet`、`ImageTraits`、`RecognitionResult` 等公共类型。
- 如果需要暴露字段，优先序列化现有公共类型的只读快照。
- 任何公共头文件字段、枚举值或 UDP 包结构变更，先提交架构合并。
