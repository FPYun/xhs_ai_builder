4. 通信、烧录与硬件验证（2 小时）
任务提示词：

你负责 CoreS3 双板通信、MATCH 稳定性、最终编译、烧录和硬件验证。

目标：
1. 双方只要都打开 MATCH，就能自动发现、连接并进入对战。
2. 玩家界面不要出现 HOST / CLIENT 概念。
3. 底层如果必须选主从，只能作为内部实现细节。
4. 任意两块板子都能配对，不依赖写死 MAC。
5. 通信断开后能自动回到 MATCHING 或重试。
6. 保持本地离线，不依赖路由器、服务器或手机 App。
7. 负责最终编译、烧录和双板实机验证。

需求边界：
- 不直接修改 BattlePetPacket。
- 如果需要新增握手包、心跳包、ACK、sessionId、状态机，先输出接口变更建议。
- 不改变宠物生成、战斗机制、UI 美术细节。
- UI 只需要提供通信状态给 UI 模块使用。
- 不引入新依赖。

建议方向：
- 设计通信状态机：DISCOVERING、PAIRING、READY、BATTLING、LOST。
- 两块板都开启 AP+STA。
- 扫描到 M5PET 设备后自动协商内部角色。
- 使用心跳、超时、重试、重复包过滤提高稳定性。
- 屏幕只显示 MATCHING、CONNECTED、BATTLING、RETRYING 等玩家可理解状态。
